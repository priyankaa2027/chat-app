Building a serverless real-time chat application using AWS Lambda, Amazon API Gateway, and DynamoDB is a solid use case that eliminates the need for managing infrastructure. Here's an outline of how you can design and implement this system:

🔧 Architecture Overview

Components:

  • Frontend: Web UI using HTML/CSS/JavaScript (React optional).

  • API Gateway: Exposes REST endpoints or WebSockets.

  • Lambda Functions: Business logic.

  • DynamoDB: Stores chat messages and user info.

  • WebSocket support (optional): For real-time updates.

  • (Optional): AWS Cognito for authentication.

📐 Architecture (REST-Based)

🔌 API Endpoints (via API Gateway + Lambda)

1. POST /sendMessage

  • Input: { chatRoomId, senderId, message }

  • Action: Lambda stores the message in DynamoDB.

2. GET /getMessages?chatRoomId=room1

  • Action: Lambda fetches last N messages from DynamoDB.

3. GET /users

  • Action: Lambda returns active users.

🔄 Real-time Updates (Optional)

To make it real-time, use WebSocket APIs with API Gateway.

WebSocket Routes:

  • $connect: Authenticate and store connection.

  • $disconnect: Remove connection.

  • sendMessage: Trigger Lambda to write to DB and push to all connected clients via their connectionId.

Store all connectionIds in a separate DynamoDB table for active users.

🧠 Lambda Sample (sendMessage.js)

javascript
const AWS = require('aws-sdk');
const docClient = new AWS.DynamoDB.DocumentClient();
const TABLE_NAME = "ChatMessages";

exports.handler = async (event) => {
    const body = JSON.parse(event.body);
    const { chatRoomId, senderId, message } = body;

    const timestamp = new Date().toISOString();

    const params = {
        TableName: TABLE_NAME,
        Item: {
            chatRoomId,
            timestamp,
            senderId,
            message
        }
    };

    await docClient.put(params).promise();

    return {
        statusCode: 200,
        body: JSON.stringify({ status: 'Message sent' }),
    };
};

✅ Step 3: Create Another Lambda (getMessages)

js
const AWS = require('aws-sdk');
const docClient = new AWS.DynamoDB.DocumentClient();

exports.handler = async (event) => {
    const chatRoomId = event.queryStringParameters.chatRoomId;

    const params = {
        TableName: "ChatMessages",
        KeyConditionExpression: "chatRoomId = :id",
        ExpressionAttributeValues: {
            ":id": chatRoomId
        },
        ScanIndexForward: false, // latest messages first
        Limit: 20
    };

    const data = await docClient.query(params).promise();

    return {
        statusCode: 200,
        body: JSON.stringify(data.Items),
    };
};

✅ Step 5: Frontend (HTML/JS Example)

html
<form id="chatForm">
  <input type="text" id="senderId" placeholder="Your Name" />
  <input type="text" id="message" placeholder="Message" />
  <button type="submit">Send</button>
</form>

<div id="chatBox"></div>

<script>
const sendMessage = async () => {
  const senderId = document.getElementById("senderId").value;
  const message = document.getElementById("message").value;

  await fetch("YOUR_API_ENDPOINT/sendMessage", {
    method: "POST",
    body: JSON.stringify({
      chatRoomId: "room1",
      senderId,
      message
    })
  });

  document.getElementById("message").value = "";
};

const getMessages = async () => {
  const res = await fetch("YOUR_API_ENDPOINT/getMessages?chatRoomId=room1");
  const messages = await res.json();

  const chatBox = document.getElementById("chatBox");
  chatBox.innerHTML = messages
    .map(msg => `<p><strong>${msg.senderId}</strong>: ${msg.message}</p>`)
    .join("");
};

document.getElementById("chatForm").addEventListener("submit", e => {
  e.preventDefault();
  sendMessage().then(getMessages);
});

setInterval(getMessages, 3000); // Poll every 3s
</script>

Replace YOUR_API_ENDPOINT with the API Gateway invoke URL
